# Завтракаст СДВГ

«Синдром Дефицита Где Голубь» — русскоязычный статический плеер случайных тем
и законченных минут из разных лет подкаста «Завтракаст».

## Требования

- Node.js `>=22.13.0`

## Быстрый старт

```bash
npm install
npm run dev
```

Приложение не имеет API, базы данных и авторизации. Подготовка архива,
транскриптов и редакционной разметки выполняется отдельными build-time
командами; в браузере загружаются только статические JSON и удалённые MP3.

В репозитории уже зафиксирована воспроизводимая выборка из 12 выпусков — по
одному на каждый год с 2015 по 2026. Текущий playback-каталог содержит первый
полностью подготовленный пилотный выпуск №2: 19 тем, 88 законченных минут,
полный синхронизированный транскрипт и WebVTT-субтитры. Остальные локальные
аудиофайлы и промежуточные результаты находятся в игнорируемой `.cache`.

## Подготовка данных

Переменные Whisper можно скопировать из `.env.example` или передать в оболочке:

```bash
export WHISPER_API_BASE=http://192.168.3.21:12017
export WHISPER_MODEL=large-v3-turbo-q5_0
```

Пайплайн выполняется по этапам и безопасно продолжается после прерывания:

```bash
npm run data:select:check
npm run data:verify-audio
npm run data:fetch
npm run data:split
npm run data:transcribe -- --all
npm run data:normalize -- --all
npm run data:validate -- --all
npm run data:build -- --all
```

MP3 используются только как локальный build-time кэш и никогда не попадают в
Git или PWA Cache Storage. В приложении аудио воспроизводится с официальных
URL Завтракаста.

Редакционная разметка хранится в
`data/episodes/<id>/editorial.json`. Тема задаётся целыми границами фраз, а
intro, музыка, реклама, донаты, благодарности и технические паузы отмечаются
как исключения. Валидатор не пропускает пересечения, произвольные таймкоды и
минутные фрагменты вне диапазона 45–75 секунд.

## PWA

После первого успешного открытия приложение устанавливается и продолжает
показывать оболочку, локальные обложки, транскрипты и субтитры без сети.
Удалённые MP3 намеренно работают только онлайн.

## Проверки

- `npm run lint`: проверить код линтером
- `npm run typecheck`: проверить TypeScript
- `npm test`: собрать приложение и запустить тесты
- `npm run test:e2e`: запустить браузерные сценарии на dev-сервере
- `npm run test:e2e:production`: собрать приложение и проверить production-PWA
  на desktop и mobile
