# Подготовка аудио и транскриптов

Пайплайн выполняется только локально во время подготовки релиза. MP3 не
попадают в приложение и Git; в runtime плеер использует официальные URL из
метаданных.

## Требования

- Node.js 22.13 или новее;
- `ffmpeg` и `ffprobe` в `PATH`;
- OpenAI-совместимый Whisper API.

Скопируйте параметры из `.env.example` в окружение или передайте
`--api-base` и `--model`. Рекомендуемая модель —
`large-v3-turbo-q5_0`.

Каждая дорогая команда требует явного `--episode <id>` либо `--all`, чтобы
случайно не запустить обработку всех 33+ часов аудио.

Только локальные обложки без MP3 можно получить командой
`npm run data:fetch -- --all --covers-only`.

## Один выпуск

```bash
npm run data:fetch -- --episode zc-02
npm run data:split -- --episode zc-02
npm run data:transcribe -- --episode zc-02
npm run data:normalize -- --episode zc-02
```

Для проверки одного запроса без запуска всего выпуска используйте
`npm run data:transcribe -- --episode zc-02 --chunk 0`.

`data:fetch` проверяет полноту ответа и декодируемость MP3. `data:split`
находит тишину около целевых 12-минутных границ и создаёт mono PCM WAV 16 kHz.
Манифест хранит абсолютные границы и SHA-256 каждого чанка.

`data:transcribe` отправляет чанки строго последовательно. Каждый валидный
ответ атомарно сохраняется в `.cache/whisper/<episode>/`; повторный запуск
пропускает результаты только при совпадении SHA чанка, модели, prompt и версии
пайплайна. Сетевые ошибки, 408, 429 и 5xx повторяются до трёх раз; постоянные
4xx не повторяются.

`data:normalize` прибавляет абсолютные смещения и объединяет только целые
Whisper-сегменты до конечной пунктуации. В `transcript.json` каждая фраза
содержит диапазон исходных сегментов; поэтому валидатор может доказать, что
таймкод не создан внутри сегмента и текст не потерян. Незаконченный хвост
сохраняется с `complete: false` и не используется как граница минутного
фрагмента.

Все временные MP3, WAV и сырые ответы находятся в `.cache/` и игнорируются
Git. Коммитятся только локальные обложки и нормализованные транскрипты.
