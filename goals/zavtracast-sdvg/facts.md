# Facts

- Приложение называется «Завтракаст СДВГ — Синдром Дефицита Где Голубь» и весь пользовательский интерфейс первой версии выполнен на русском языке.
- Первая версия является полностью статическим SPA без серверной части, аккаунтов, облачной синхронизации, лайков, плейлистов, административного интерфейса и автоматического обновления RSS в браузере.
- Стартовый набор содержит 36 выпусков Завтракаста: по одному случайно выбранному выпуску каждого календарного года с 2015 по 2026 и 24 дополнительных выпуска из архива; выбранные выпуски зафиксированы в данных проекта.
- Для каждого стартового выпуска сохранены номер, название, дата, год, длительность, ссылка на страницу, URL обложки, удалённый URL MP3, полный транскрипт с таймкодами фраз и темы с начальным и конечным таймкодами.
- В проекте есть воспроизводимый build-time пайплайн, который получает метаданные и MP3 выпусков, отправляет аудио в OpenAI-совместимый локальный Whisper API и сохраняет транскрипты с временными сегментами; адрес API и модель настраиваются, а рекомендуемая модель — large-v3-turbo-q5_0.
- Тематическая разметка создаётся из шоунотов и транскрипта с помощью Codex, хранится в редактируемом JSON и проходит автоматическую проверку диапазонов, порядка таймкодов и ссылок на существующие фразы.
- Заставки, музыка, реклама, донаты, длинные списки благодарностей и технические паузы помечены как исключённые диапазоны и не попадают ни в тематическую, ни в минутную очередь.
- Пользователь может в любой момент переключать режим воспроизведения между «Темами» и «Минутой», а выбранный режим сохраняется локально.
- В режиме «Темы» приложение выбирает целую содержательную тему из любого подготовленного года, начинает воспроизведение с её первого таймкода, останавливает на последнем и автоматически переходит к следующей случайной теме.
- В режиме «Минута» приложение выбирает фрагмент на границах предложений длительностью максимально близко к 60 секундам и в пределах 45–75 секунд, затем автоматически переходит к следующему фрагменту.
- Внутри каждого режима элементы не повторяются до исчерпания очереди; история и положение в очереди сохраняются локально, доступны кнопки «Назад» и «Следующий», а после исчерпания создаётся новая перемешанная очередь.
- Первое воспроизведение запускается только после действия пользователя в соответствии с ограничениями браузера; после этого переходы между элементами очереди происходят автоматически, пока пользователь не поставит плеер на паузу.
- Плеер показывает прогресс внутри текущего фрагмента и предоставляет воспроизведение/паузу, переход назад и вперёд, перемотку и регулировку громкости.
- Под плеером синхронно показывается текущая фраза транскрипта; из информации о выпуске открывается полный транскрипт, и нажатие на таймкод переводит воспроизведение к соответствующей фразе.
- Информация об источнике по умолчанию компактно скрыта и открывается отдельной кнопкой; она содержит тему или описание фрагмента, номер и название выпуска, дату, год, обложку и ссылку на оригинальную страницу.
- Приложение устанавливается как PWA; оболочка, обложки и текстовые данные доступны офлайн после первого открытия, но MP3 всегда воспроизводятся по сети и не скачиваются для офлайн-прослушивания.
- При отсутствии сети или недоступности MP3 приложение сохраняет управление, явно сообщает о проблеме и позволяет повторить попытку либо перейти к следующему элементу без потери очереди.
- Интерфейс адаптирован для телефона и десктопа, управляется клавиатурой, имеет заметный фокус, корректные подписи элементов и уважает системную настройку уменьшения движения.
- Визуальный стиль напоминает самостоятельный ночной радиоприёмник: графитовый фон, тёплый жёлтый акцент, голубино-синие детали, крупная типографика и ироничный образ голубя без буквального копирования сайта Завтракаста.
- Проект содержит автоматические проверки схемы данных, алгоритмов очереди и минутных границ, поведения плеера, PWA-манифеста и ключевых пользовательских сценариев; production-сборка завершается без ошибок.
